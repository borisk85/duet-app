# Дуэт — AI-эксперт по гастрономическим напиткам

## Концепция

Мобильное приложение — **AI-эксперт по гастрономическим напиткам** в кармане. Подбирает идеальный напиток к любому блюду, объединяя экспертизу сомелье в винах, бартендера в коктейлях и крепком алкоголе, и сирвейона в пиве.

Пользователь вводит что готовит или ест → приложение мгновенно подбирает конкретный напиток с объяснением почему именно он.

Не просто "красное вино", а "Мальбек средней выдержки, полнотелый, подчеркнет жирность мяса".

Все подборки сохраняются в истории и избранном — ничего не теряется в отличие от чатов с ChatGPT.

**Важно по позиционированию:**
- ❌ НЕ называть приложение "сомелье" — это сужает до вина (а вино = ~30% функционала)
- ❌ НЕ использовать слово "алкоголь" в публичных текстах — звучит клинически и триггерит anti-alcohol аудиторию
- ✅ Использовать "**гастрономические напитки**" — премиально, охватывает все категории, исключает чай/кофе/воду
- ✅ Вино остаётся флагманом по умолчанию (для большинства блюд это лучший выбор), но через **функциональную глубину**, а не через позиционирование

## Проблема которую решает

Человек хочет удивить на ужине, подобрать напиток к блюду, но:
- Не разбирается в напитках и винных картах
- В магазине консультанта нет, вин и виски — тысячи видов
- Гуглит, спрашивает в чатах — ответы теряются
- В итоге берет первое попавшееся или вообще ничего

Приложение заменяет консультанта, запоминает предпочтения, помогает прийти в магазин с готовым названием.

## Охват (не только вино)

- Вино (красное, белое, розовое, игристое)
- Крепкий алкоголь (виски, коньяк, водка, джин, ром, текила)
- Коктейли и их закуски
- Пиво к еде
- Безалкогольные альтернативы (опционально)

## Целевая аудитория

- Женщины 28-45 — вино с подругами, романтический ужин
- Белые воротнички — бизнес-ужины, подарить бутылку, произвести впечатление
- Люди которые хотят выглядеть разбирающимися но не разбираются
- Рынок СНГ в приоритете (русский язык, локальные магазины)

Поколение Z — не целевая аудитория (тренд на снижение потребления алкоголя).

## Ключевые экраны

1. **Главный** — два режима поиска (переключатель):
   - Еда → Алкоголь (основной): подробное описание блюда → ТОП-3 напитка
   - Алкоголь → Еда (обратный): выбрал напиток → что закусить
2. **Результат** — ТОП-3 варианта, разбитые по категориям (вино / крепкий / пиво / коктейль), с маркой, объяснением и бюджетом. Кнопки "Сохранить в избранное" и "Купить"
3. **Избранное** — сохранённые дуэты, структурированные по типу алкоголя
4. **История** — все подборки за последние 30 дней, потом автоудаление
5. **Профиль** — предпочтения по алкоголю (влияют на приоритет в результатах), подписка, регион

## Логика подбора

- Направление: еда → алкоголь (основное), алкоголь → еда (дополнительное)
- Описание блюда должно быть подробным: не "стейк", а "говяжий стейк", не "рамен", а "рамен с курицей"
- AI выдаёт ТОП-3 варианта с учётом: типа блюда, бюджета пользователя, региона (РФ/Казахстан/другое), предпочтений из профиля
- Результат разбит по категориям алкоголя, не один ответ
- Предпочтения пользователя из профиля влияют на приоритет категорий в выдаче

## Обработка ошибок API (обязательно)

| Ситуация | Что показываем |
|----------|---------------|
| Claude API недоступен (5xx) | "Сервис временно недоступен. Попробуйте через минуту." + кнопка Повторить |
| Таймаут (>30 сек) | "Долго думаем... Попробуйте ещё раз" |
| Rate limit (429) | "Достигнут дневной лимит. Обновите до Premium для безлимита." |
| Нет интернета | "Нет подключения к сети" |
| Пустой ввод | Inline-ошибка под полем, не снекбар |

Все ошибки — SnackBar с цветом `Colors.red.shade800`, floating, скруглённый.

## Стандарты разработки (Best Practices)

Всегда строить как топовые монетизированные мобильные приложения:
- **Навигация:** Bottom Navigation Bar (3-5 вкладок), никаких бургер-меню
- **Загрузка:** Skeleton screens вместо спиннеров
- **Анимации:** Плавные переходы между экранами (Hero, Fade, Slide)
- **Состояния:** Всегда обрабатывать loading / error / empty state
- **Тапы:** Минимальная зона касания 48px
- **Сетка:** Отступы кратны 8px (8, 16, 24, 32...)
- **Обратная связь:** Haptic feedback на ключевых действиях
- **Шрифты:** Не больше 2 начертаний на экране
- Перед использованием любой библиотеки — проверять документацию через context7

## Инструкции по взаимодействию с владельцем проекта (Boris)

### Автономность

**Все что Claude может сделать сам — делает сам, не спрашивая.**
Код, правки, конфигурация, PATH, файлы, структура — все делается без вопросов.

Boris делает только то, что Claude физически не может: нажать кнопку в UI, ввести пароль, пройти авторизацию в браузере.

### Когда менять код

**НИКОГДА не вносить изменения без явной команды.**
Если Boris описывает проблему, показывает что не так, или говорит "это хуйня" — только подтвердить понимание и предложить решение. Ждать команды.

Команда на изменение = явное "сделай", "меняй", "правь", "исправь", "убери", "поменяй".
Без явной команды — только обсуждение, никакого кода.

### Перед тем как говорить "не реализовано"

Всегда сначала проверять код через grep/read. Никогда не говорить "нужно сделать X" без проверки что X уже не существует в кодовой базе.

### Стиль общения

- Обращаться к Boris на **"ты"** — никогда не на "вы"
- Никогда не говорить "спокойной ночи", "отдыхай", "иди спать" — Boris сам знает когда ему спать
- Не объяснять очевидные шаги которые уже делали: "нажми Deploy", "подожди пока задеплоится", "вставь код в редактор" — он знает
- Не переспрашивать "ты уже сделал X?" если только что подтвердил — доверять ответам
- Если что-то не работает — искать причину в инструменте или коде, не в действиях Boris

### Что никогда не предлагать при проблемах с сайтом

- Ctrl+Shift+R (жесткое обновление)
- Очистка кеша Cloudflare
- Открыть в инкогнито
- Любые браузерные трюки

Диагностировать реальную причину (код, деплой, DNS).

### Git и деплой

После каждого `git commit` бэкенда — сразу `git push origin main`.
Не спрашивать "деплоить?" — деплоить сразу.

### Анализ фото

Коротко: 2-3 предложения, суть + главная деталь. Без списков и пунктов.

### Копирайтинг (маркетинговые тексты)

Писать живо, как человек. Конкретика вместо общих слов: "за 5 минут", "$0/мес навсегда" вместо "начать можно прямо сейчас". Капслок для акцентов. Перед отправкой прочитать вслух — если звучит как робот, переписать.
